- Interface, 2021, No.1

GPU版のプログラムも提供されているが，
opencvをソースからビルドする必要があり難しそうなので
ここでは大人しくCPU版を活用する．

GPU版のプログラムをざっとみた限り，
プログラム自体はそこまで難しくなさそう．
基本的にGPUへデータを転送して，GPUで処理して，CPUに返すというもの. 
関数もプレフィクスとして`cuda`がつくだけ.

## エッジを抽出する

### 画像深度について
たとえば, CV_8Uにおいて8はビット数を表し，Uは符号なし整数であることを表す。
8UC1も同じものでチャンネル数が１ということを表すが，
一般にチャンネル数が１を表すC1は省略して表す。

### エッジ検出について
大きくは次の流れ

- 画像の平滑化
- 画素値の勾配
- エッジの検出
    - 線画の描画

## 膨張と収縮

白黒画像に対して単純に「膨張・収縮を行うことで得られる結果」を利用する手法をモルフォロジー変換と呼ぶ。
通常は２値画像に対して行う変換ですが，カラー画像に対して変換を行うと，モザイク画を生成できる。

### クロージング

画像の白い部分を指定回数膨張させた後に，同じ回数収縮させる処理で適切回数を処理すると，黒ノイズ除去に有効な処理とんらう。

### オープニング

クロージングの逆

### 勾配

白い箇所の膨張と収縮を繰り返した画像の差分を求める処理。
輪郭線を得ることが可能である。

### トップハット/ブラックハット
明るい部分・暗い部分を強調する.
